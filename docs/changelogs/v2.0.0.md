# v2.0.0 — 2026-04-11

The major-version cut from v1.x. 22 commits between v1.12.0 and v2.0.0, focused on the `@v1`→`@v2` ref bump, GitHub App authentication migration, the Node 24 runtime adoption, and a new coverage-badge feature for PRs.

## Breaking changes

- **All internal `CLDMV/.github/.github/...@v1` refs bumped to `@v2`** (`7fcb6c8`, `e6e6c2f`). Consumer templates in `examples/individual-repo-workflows/` updated in the same release. Existing `@v1` pinning continues to work against the frozen v1 line.
- **GitHub App authentication migrated to `client-id`** (`2f04247`). Workflows now reference `BOT_APP_CLIENT_ID` rather than `BOT_APP_ID`. Org secret name standardized to `CLDMV_BOT_APP_CLIENT_ID` / `CLDMV_BOT_APP_PRIVATE_KEY`.

## Added

- **Coverage badge injected into PR descriptions** (`5ea4cd7`, `8349489`) — on pull-request events, the CI workflow patches a Shields.io-style badge + breakdown table into the PR body. No files committed; replaces the older "comment on every push" pattern.
  - `enable_coverage_pr_comment` input (default `true`); `for-the-badge` style with vitest logo (`e53d4d2`).
  - Gated to push-to-default for badge-branch publishing (`ec8b33c`) and `default_branch` wired through `workflow_dispatch` (`ded6f2a`).
- **`update-existing-pr` job** in the release workflow — re-runs the bump on the existing release PR rather than opening a duplicate, including title refresh (`e36a2b7`) and package.json bump + version commit (`80626b8`).
- **Setup + dry-run + update-major-version-tags guides** under `examples/` (`4ed28ce`, `1c54320`).

## Changed

- **Coverage badge push gated to default branch only** (`ec8b33c`) — feature branches no longer publish stale coverage to the `badges` branch.
- **`feature:` accepted as a `feat:` alias** in commit detection (`7bbac66`).
- **Type-check command updated** for the CI workflows (`a8d22c7`).
- **Generated files cleaned before badges-branch checkout** to avoid a dirty-tree push failure (`c6fb3ee`, `e97c959`).
- **`ci.yml` example uses `github.token` directly** instead of indirecting through a `GH_TOKEN` secret (`c212e65`).
- **Workflow indentation reformatted** uniformly (`e669990`).

## Fixed

- **Release workflow waits for tag-ref propagation** before creating the GitHub Release (`915c377`) — race between push and release create that occasionally produced "release without tag" entries.
- **Version-bump logic corrected for re-runs** (`54ee977`) — bump no longer doubles when a release PR is rebased.
- **`src/` restored before bump commit** in the release flow (`7bbac66`) — release commits no longer accidentally drop unstaged source changes.
- **PR label sync** under the App `client-id` flow (`2f04247`).

## v2.x patch series (v2.0.1 → v2.0.39)

Thirty-nine patch releases shipped on the v2.0 line. Major themes:

- **Tag-health auth hardening** — multiple iterations on `extraheader` handling, App-token vs Actions-token usage, and rolling-tag push specifics.
- **Org label sync robustness** — alias resolution, archived/disabled-repo summary, ENOENT in external private repos, 2026-03-10 API schema fixes.
- **Versioning robustness** — anchor commit ranges to merge-base, dedup bumps, guard against bot's own bump commits.
- **`lts_only_matrix` input** added to CI / release / publish (foreshadowed the v3 `min_node_version`-empty semantics).

Each patch is documented in its GitHub Release.
