# v4.1.0 — 2026-05-22

Minor release adding bot-bypass defaulting to the ruleset generator. Single-feature cut.

## Added

- **Ruleset generator pre-adds the bot App to the `next` and `hotfixes` bypass lists by default.** The `next`/`hotfixes` integration branches need the CLDMV bot in their ruleset bypass actor list — without it, the persistent release PR machinery is rejected when it tries to push the `chore: bump version` commit or reset the branch. v4.0.x required adopters to add the bot manually after generating rulesets, which was an easy step to miss. v4.1.0 makes the generator emit rulesets with the bot already in the bypass list (opt-out via a checkbox in the generator UI). `master` does NOT get the default — bot bypass on master would defeat the require-PR rule. (feat: c4ec476)

## Changed

- Generator UI gains a `Bot App in bypass` toggle (default ON for `next`/`hotfixes`, hidden for `master`); rulesets exported via the generator carry the bypass entry as needed. (`docs/tools/ruleset-generator/`)
- `data/rulesets/next.json` and `data/rulesets/hotfixes.json` updated to reflect the new default.

## Migration note

Existing v4 adopters already configured their rulesets manually during the v4.0.x adoption window — re-importing from v4.1.0 is optional. New adopters skip the manual bypass step.
