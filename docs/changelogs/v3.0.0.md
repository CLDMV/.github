# v3.0.0 ‚Äî 2026-05-20

First major release covering 49 commits since v2.0.39. Adds a comprehensive security + automation suite, formalizes the dogfood architecture for testing workflows against their own PRs, introduces a CLA bot, branch retention with naming convention, release notifier, bundle-size diff, gh-pages docs publisher, master commit audit, and 12 new copy-paste templates for consumers.

See also: [migration guide for v2 ‚Üí v3 consumers](../migration/v2-to-v3.md).

## Breaking changes

- **All consumer workflow refs must move from `@v2` to `@v3`.** Internal `CLDMV/.github/.github/{workflows,actions}/...@v3` references are updated repo-wide. Consumer templates in `examples/individual-repo-workflows/` already point at `@v3`. Existing `@v2` consumers continue to work against the v2 line; nothing forces an upgrade.
- **`min_node_version` empty now means "no matrix"** (closes [#2](https://github.com/CLDMV/.github/issues/2)). The CI / publish workflows previously created a matrix anyway when `min_node_version` was an empty string. Now an empty value means run only `max-node-major` + `lts/*` as a single check. To opt back into the matrix, set `min_node_version: "20"` (or your floor) explicitly.
- **`publish.yml` triggers ONLY on `push: branches: [master, main]`** (closes [#1](https://github.com/CLDMV/.github/issues/1)). Previously a leftover trigger let publish run from PR branches, producing the silent-regression bug where master's content could be skipped. Now publish is exclusively a master/main affair.
- **CodeQL `paths` / `paths-ignore` move from `init@v3` inputs to a `config-file:`.** The upstream `github/codeql-action/init@v3` doesn't accept these as direct inputs. `reusable-codeql.yml` now generates a temp `codeql-config.yml` from the `paths` / `paths_ignore` CSV inputs and passes it via `config-file:`. Consumers using the reusable transparently ‚Äî no template change needed.

## Added

### Security baseline

- `reusable-codeql.yml` ‚Äî CodeQL SAST. Comma-separated `languages` input now round-trips through the new `common/utilities/parse-csv-list` action because GitHub Actions has no `split()` builtin. (Batch 6.3)
- `reusable-dependency-review.yml` ‚Äî PR-time CVE diff via GitHub Advisory DB.
- `reusable-container-scan.yml` ‚Äî Trivy vulnerability scanner; plugs into the docker-publish flow.
- Scorecard wrapper for OpenSSF Scorecard publishing to the public scoreboard. (Batch ‚Äî security)

### Automation suite

- `reusable-stale.yml` ‚Äî daily inactive-issue / inactive-PR sweep. Roll-our-own (no `actions/stale` dep). (Batch 2.3)
- `reusable-dependabot-auto-merge.yml` ‚Äî approves + auto-merges patch/minor Dependabot bumps after CI. (Batch 3.2)
- `reusable-pr-labeler.yml` ‚Äî path-based PR labels feeding the existing label catalog; per-repo override via `.github/labeler.yml`. (Batch 5.2)
- `reusable-welcome.yml` ‚Äî first-PR / first-issue welcome with conditional CONTRIBUTING / CLA / COC links. (Batch 5.3)
- `reusable-branch-retention.yml` ‚Äî on merge, prunes most head branches; keeps last N of `release/*` / `hotfix/*`. (Batch 1.3)
- `reusable-cla.yml` ‚Äî per-PR CLA bot with org-member exemption; "I have read the CLA" comment signs. (Batch 1.1)

### Packaging / docs

- `reusable-bundle-size.yml` ‚Äî PR-time `dist/` size diff (raw / gzip / brotli) for runtime libs. (Batch 5.4)
- `reusable-docs-publish.yml` ‚Äî builds docs and pushes to `gh-pages` as a signed bot commit. (Batch 6.1)
- `reusable-release-notifier.yml` ‚Äî on `release: published`, fans out to Discord / Slack / generic webhooks. Per-repo channel config merged with org defaults. (Batch 6.2)

### Release-flow companions

- `workflow-sync-open-release-prs.yml` ‚Äî when any PR merges to master, fan-out refreshes every open release PR's version + changelog. Closes the silent-regression gap from P3.1/P3.2.
- `local-master-commit-audit.yml` + `audit-commit-subject` action ‚Äî files an Issue if a master commit doesn't match the expected release-flow subject pattern. (Batch 5.1)

### Dogfood (`local-*.yml`) workflows

- `local-ci.yml` ‚Äî Node syntax check, YAML parse, actionlint, stale @v2 detector. Each check appends a section to `$GITHUB_STEP_SUMMARY` so failures are visible on the run page without drilling into step logs.
- `local-codeql.yml`, `local-tag-health.yml`, `local-stale.yml`, `local-labeler.yml`, `local-welcome.yml`, `local-branch-retention.yml`, `local-master-commit-audit.yml`, `local-release.yml`, `local-sync-release-prs.yml`, `local-update-major-version-tags.yml` ‚Äî call reusables via **relative** `./.github/workflows/reusable-X.yml` so PRs test against the PR's version of the reusable, not the published `@v3` tag.

### Org conventions

- `data/github-labels.json` gains 5 new labels for the new automations.
- `docs/conventions/branch-naming.md` documents the branch-name policy (`master`, `release/*`, `hotfix/*`, `feat/*`, `fix/*`, etc.) backed by `reusable-branch-retention.yml`.
- `scripts/setup-org-rulesets.mjs` ‚Äî installer for the org-wide branch-naming Ruleset.
- `CLA.md` ‚Äî contributor license agreement text consumed by `reusable-cla.yml`.

### Examples reorganization

- `examples/individual-repo-workflows/` regrouped into five category subfolders (`core-cicd/`, `release-companions/`, `security/`, `automation/`, `packaging-docs/`).
- `examples/guides/` consolidates `WORKFLOW-SETUP-GUIDE.md`, `DRY-RUN-GUIDE.md`, and `UPDATE-MAJOR-VERSION-TAGS-GUIDE.md`. The setup guide now covers all 21 templates with per-template sections (was 5 in v2).

## Changed

- **Marketplace action versions bumped** (Node 20 deprecation): `actions/checkout` v4 ‚Üí v6, `actions/setup-node` v4 ‚Üí v6, `actions/upload-artifact` v4 ‚Üí v6, `actions/download-artifact` v4 ‚Üí v7. `actions/dependency-review-action` left at v4 (no v5 yet).
- **`local-ci.yml` actionlint pin bumped from 1.7.7 ‚Üí 1.7.12** ‚Äî the first release that recognizes `using: node24` (eliminates a class of false-positive findings against this repo's Node actions).
- **`reusable-tag-health.yml` outputs dropped dead `||` fallback halves** that referenced job IDs (`update-version-tags`, `fix-bot-signatures`) renamed away under the `unified-*` consolidation.
- **API version aligned to `2026-03-10`** repo-wide (and a dead debug job removed). The Bearer / `Accept: application/vnd.github+json` / `X-GitHub-Api-Version` convention was already enforced in v2.x; this cut catches a few stragglers.
- **`workflow-release.yml` calculates version against current master**, not the merge-base, so parallel release PRs can't silently regress past each other.
- **`workflow-publish.yml` skips publish/release jobs when version is unchanged on npm** ‚Äî re-running the workflow on the same SHA is now idempotent.

## Fixed

- **#1** ‚Äî publish was triggering from PR branches (see Breaking changes).
- **#2** ‚Äî empty `min_node_version` was still creating a matrix (see Breaking changes).
- **`reusable-codeql.yml`** ‚Äî replaced the nonexistent `split()` expression (`fromJson(format('["{0}"]', join(split(inputs.languages, ','), '","')))`) with a setup job using a `parse-csv-list` action; matrix now resolves cleanly.
- **`local-ci.yml`** ‚Äî pre-existing actionlint findings across `reusable-codeql.yml`, `reusable-tag-health.yml`, `local-codeql.yml`, `workflow-docker-publish.yml` were buried in step logs. Refactor surfaces them on the run summary page and the underlying bugs are fixed.
- **`local-ci.yml @v2` self-grep** ‚Äî pattern was matching `local-ci.yml`'s own step names and error messages; excluded the file from its own scan.

## Removed

- The dead `update-version-tags` and `fix-bot-signatures` job-ID references in `reusable-tag-health.yml` outputs (consolidated under `unified-*` long ago, but the dead `||` fallbacks lingered).
